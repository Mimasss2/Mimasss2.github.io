git cheatsheet

A git command cheatsheet.

1 Basics

  • git init 初始化当前目录为git仓库
  • git help <command> 查看某个命令的帮助文档
  • git status 查看当前状态
  • git add <filename> 将某个指定的文件加入暂存区
    • git add . 将当前目录下的所有文件加入暂存区
  • git commit 创建一个新的版本（commit）
    • -m 指定message
  • git log 显示历史版本
  • git log --all --graph --decorate 用DAG可视化历史版本
  • git diff 对比修改前后文件的变化
    • git diff <filename> 默认将此刻的文件与HEAD版本的对比
    • git diff <revision> <filename> 将此刻的文件与指定版本（revision）的对比
    • git diff <revision1> <revision2> <filename 将指定版本（revision2）的文件与指定版本（revision1）的对比
  • git checkout <revision> 切换到指定版本（commit）

注：<revision> 可以使branch，也可以是某次commit的hash code，也可以是HEAD

2 分支与合并

  • git branch 查看所有分支
  • git branch <name> 创建新分支
  • git checkout -b <name> 创建并切换至新分支
  • git merge <revision> 将指定的版本（revision）合并到当前分支
  • git mergetool 使用高级的工具解决合并冲突，需要提前设置，比如vimdiff
  • git rebase 将某个分支合并到base

3 与远程仓库互动

  • git remote 列出所有的远程仓库
  • git remote add <name> <url> 添加远程仓库，由url指定地址，并将其记为<name>
  • git push <remote> <local branch>:<remote branch> 将指定的本地分支，推送到指定的远程仓库的指定分支
  • git branch --set-upstream-to=<remote>/<remote branch> 设置某个分支的上游仓库/分支，设置后可直接执行git push 而无需再指定后面的一长串
  • git fetch 从远程拉取内容
  • git pull 从远程拉取内容并合并，等同于git fecth; git merge
  • git clone 从远程下载仓库
    • git clone <url> 下载指定url的repo
    • git clone <url> <subfolder> 下载指定url的repo，并存到subfolder文件夹

4 撤回操作

  • git commit --amend 修改某次commit的内容/message
  • git reset HEAD <file> 将某个文件恢复到HEAD的版本
  • git checkout -- <file> 撤销某个文件的改动

5 高级

  • git config 定制化config，也可直接查看.gitconfig
  • git clone --depth=1/ --shallow 只clone最近1次commit，防止commit记录太长耗费空间
  • git add -p 交互式将文件加入暂存区，可指定保留/放弃某几行的更改
  • git rebase -i 交互式rebase
  • git blame 查看谁最后一次修改某一行，此修改对应哪个commit
  • git stash 暂时将修改从当前目录的文件中移除，可使用git stash pop恢复
  • git bisect 二分查找历史记录
  • .gitignore 指定不被git跟踪的文件

6 其他

  • git cat-file -p <revison> 指定对应编号的object的内容
